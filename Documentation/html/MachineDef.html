Machine Definition

The memory map of the simulated machine is controlled by a file called machine.xml which is in the same directory as the application. The file is in XML format.

Here is a sample file:


<machine>

<device type='RAM' name='Main RAM'>
<map start='0000' length='2000' />
</device> <device type='ROM' name='Main ROM'>
<map start='8000' length='8000' />
</device>

<file type='ORG'>test.bin</file>

</machine>

Tags:

<machine></machine>
The machine specification must be included between these tags

<device type='type' name='name'></device>

This tag specifies a device in the machine. A machine can have one or more devices. Each device must have 1 or more map tags.

Attributes:

type - Specifies the type of device. See below for device types

name - The name of the device

 

<map start='xxxx' length='xxxx' />

Used to specify where a device appears in memory. Each device can have one or more of these tags.

Attributes:

start - hex starting address of device

length - number of bytes of memory the device occupies in hex

<file type='type' start='xxxx'>filename</file>

Used to load a file into the emulated system. Each machine can have 0 or more of these tags. This tag MUST appear after the memory where the file is loaded into is declared. filename is the full filename of the file to load.

Attributes:

type - What type the file to load is. Supported types are ORG, RAW, and RAS.

start - Sepecifies the hex address to start loading the file into. Only needed for the RAW file type.
